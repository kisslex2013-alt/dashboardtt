# AI Conventions — Dashboardtt

> Правила и соглашения для эффективной работы с AI-ассистентами

## 📋 Выбор модели

| Задача | Модель | Причина |
|--------|--------|---------|
| Простые completions, документация | Haiku / Flash | Дёшево, достаточно |
| Генерация функций, helpers | Haiku / Flash | Хватает для шаблонных задач |
| Рефакторинг нескольких файлов | Sonnet / Pro | Требует понимания scope |
| Сложная логика, архитектура | Sonnet / Pro | Reasoning-heavy |
| Исследование новых технологий | Opus | Learning curve |

**Правило: 70% задач — дешёвые модели, 30% — мощные.**

## 🎯 Шаблон промта

```
Role: [специализация, например: React optimization expert]
File: [путь:строки или только сигнатуры]
Task: [атомарное действие — одна чёткая задача]
Context: [только необходимый минимум]
Return: diff / instruction / full (предпочтительно: diff)
Constraint: [ограничения по формату или scope]
```

### Примеры

✅ **Хорошо (300 токенов):**
```
Role: TypeScript caching specialist
File: src/services/UserService.ts:getUser()
Task: Add Redis cache layer
Pattern: check cache → query DB → cache → return
TTL: 3600s
Return: Diff format only
```

❌ **Плохо (3000+ токенов):**
```
Вот весь мой файл UserService.ts...
[полный код на 100 строк]
Нужно добавить кэширование к методу getUser...
Верни полный обновлённый файл.
```

## 🚫 Запрещённые паттерны

1. **Вставка целых файлов** — используй сигнатуры функций
2. **Чаты >5 сообщений** — начинай новый чат на каждую задачу
3. **Full-code mode для edits** — всегда используй diff
4. **Несколько несвязанных задач в одном промте** — атомарные задачи
5. **"Make my code better"** — конкретизируй что именно улучшить

## ✅ Чеклист перед запросом

- [ ] Выбрана правильная модель?
- [ ] Контекст сокращён до необходимого? (сигнатуры, не реализация)
- [ ] Задача атомарна и однозначна?
- [ ] Роль и формат вывода указаны?
- [ ] Примеры минимальны? (не более 2-3)
- [ ] Нужен ли новый чат? (если >5 сообщений — да)

## 📁 Структура контекста по уровням

```
LEVEL 1 (Критически важно):
  - Текущий файл (только сигнатуры)
  - Интерфейсы, типы, экспорты

LEVEL 2 (Важно):
  - Соседние файлы в папке
  - Зависимости
  - Типы ошибок

LEVEL 3 (Справочно):
  - Документация
  - Примеры использования
```

При приближении к лимиту: LEVEL 3 → LEVEL 2 → LEVEL 1

## 📊 Ожидаемая экономия

| Техника | Экономия |
|---------|----------|
| Сигнатуры вместо полного кода | 84% |
| Diff вместо full file | 90% |
| Новый чат на каждую задачу | 60-80% |
| Структурированные промты | 85% |

**Финальное число:** С $150-200/месяц до $30-50/месяц при том же качестве.
