2. Системные промты и настройки
2.1 Master System Prompt для визуального фронта
markdown# UI/UX Generation Guidelines

## Core Principles
1. **Visual Hierarchy First**: Every element has clear purpose and weight
2. **Spacing is Sacred**: Use consistent spacing scale (4px, 8px, 16px, 24px, 32px, 48px, 64px)
3. **Typography Scale**: Never random font sizes. Use scale (12px, 14px, 16px, 18px, 20px, 24px, 30px, 36px, 48px)
4. **Modern Color Systems**: Use HSL with proper contrast. Dark mode by default.
5. **Micro-interactions**: Subtle hover states, transitions (200ms ease)

## Tech Stack Defaults
- React + TypeScript
- Tailwind CSS (no custom CSS unless necessary)
- shadcn/ui components
- Lucide icons
- Framer Motion for animations

## Design Thinking Process
Before generating code:
1. What's the primary user action?
2. What's the visual focal point?
3. How does this fit modern UI patterns?
4. What's the emotional tone (professional/playful/minimal)?

## Quality Checklist
- [ ] Proper whitespace (never cramped)
- [ ] Consistent border-radius
- [ ] Hover states on all interactive elements
- [ ] Loading states
- [ ] Mobile-responsive (mobile-first)
- [ ] Accessible (ARIA, keyboard navigation)
- [ ] Visual feedback (button clicks, form validation)

## Anti-patterns to Avoid
- ❌ Centered walls of text
- ❌ Bright primary colors everywhere
- ❌ Comic Sans or system fonts without fallbacks
- ❌ Equally weighted elements
- ❌ No whitespace
- ❌ Inconsistent spacing

## When Creating Landing Pages
- Hero: Full viewport height, clear CTA, visual anchor (gradient/image)
- Features: 3-column grid, icons, brief copy
- Social Proof: Logos, testimonials with photos
- Pricing: Cards with clear differentiation
- Footer: Comprehensive but organized

## When Creating Dashboards
- Sidebar navigation (collapsible on mobile)
- Metrics cards with icons and trend indicators
- Data tables with sorting, search, pagination
- Charts with proper legends and tooltips
- Empty states with helpful CTAs

## When Creating Forms
- Single column on mobile, smart columns on desktop
- Labels above inputs
- Inline validation
- Disabled submit until valid
- Success states
```

## 3. Рекомендуемый Workflow

### Этап 1: Подготовка (5 мин)
1. **Соберите референсы**
   - 3-5 скриншотов похожих UI (Dribbble, Behance, реальные продукты)
   - Сохраните в проект

2. **Определите стиль**
   - Минимализм / Bold / Gradient-heavy / Brutalism?
   - Цветовая схема (1 primary, 1 accent)
   - Настроение (серьезно / playful / futuristic)

3. **Активируйте нужные MCP**
   - Web Search (для актуальных трендов)
   - GitHub MCP (если есть open-source аналог)
   - Figma MCP (если есть макет)

### Этап 2: Первая генерация (10 мин)
```
Context: I'm building [type of app] for [audience]

Style reference: [attach screenshots or describe: "like Stripe's dashboard", "like Linear's minimalism"]

Generate initial structure:
1. List all pages/views needed
2. For each page, describe:
   - Primary user action
   - Key visual elements
   - Layout structure

Then create the main page with full implementation.

Use: React, TypeScript, Tailwind, shadcn/ui, Lucide icons
```

### Этап 3: Итерации (20-30 мин)
После первой генерации **не переписывайте все**. Точечные улучшения:
```
Current issues:
- Spacing feels cramped → "Add more whitespace, use py-8 instead of py-4"
- Typography unclear → "Create clear visual hierarchy: headings should be 2x body text"
- Colors bland → "Use gradient backgrounds: from-blue-600 to-purple-600"
- No personality → "Add subtle animations on hover, use rounded-xl corners"

Improve [specific component] based on these issues.
```

### Этап 4: Полировка (10 мин)
```
Final polish checklist:
1. Add loading states to all async actions
2. Add empty states with friendly messages
3. Ensure all buttons have hover/active states
4. Check mobile responsive (test at 375px)
5. Add micro-interactions (scale on click)
6. Verify color contrast (WCAG AA minimum)
```

## 4. Типовые ошибки zero-coder'ов и решения

### Ошибка 1: "Просто сделай красиво"
**Проблема**: Модель не понимает "красиво" одинаково с вами

**Решение через MCP**:
- Используйте Screenshot Context MCP с конкретными примерами
- Промт: "Match the visual style of [attached screenshot]"

**Решение через промт**:
```
"Красиво" означает:
- Generous whitespace (padding: 32px minimum)
- Subtle shadows (shadow-lg, not shadow-2xl)
- Consistent border radius (rounded-lg everywhere)
- Limited color palette (1 primary + neutrals)
- Smooth transitions (transition-all duration-200)
```

### Ошибка 2: Принятие первого варианта
**Проблема**: Первая генерация редко оптимальна

**Решение через workflow**:
- Всегда делайте минимум 2-3 итерации
- Используйте конкретные замечания, не "сделай лучше"

**Промт-паттерн**:
```
Keep the structure, but improve:
1. [Specific visual issue] → [Specific solution]
2. [Another issue] → [Solution]

Change only these aspects, maintain everything else.
```

### Ошибка 3: Игнорирование компонентных библиотек
**Проблема**: Модель пишет UI с нуля вместо использования shadcn/ui

**Решение через MCP**:
- Подключите Shadcn Reference MCP
- В системном промте: "ALWAYS check if shadcn/ui has this component first"

**Промт-паттерн**:
```
Before implementing [component], check:
1. Does shadcn/ui have this? (Button, Card, Dialog, etc.)
2. Can I compose existing components?
3. Only if no - create custom

Use shadcn/ui's [specific component] for this.
Ошибка 4: Нет дизайн-системы
Проблема: Каждый компонент выглядит по-разному
Решение через MCP:

Создайте design-tokens.json и подключите через File System MCP
Модель будет читать константы

design-tokens.json:
json{
  "spacing": {
    "xs": "0.5rem",
    "sm": "1rem",
    "md": "1.5rem",
    "lg": "2rem",
    "xl": "3rem"
  },
  "borderRadius": {
    "default": "0.5rem",
    "large": "0.75rem"
  },
  "colors": {
    "primary": "221 83% 53%",
    "accent": "142 76% 36%"
  }
}
```

**Промт**:
```
Use design tokens from design-tokens.json for all spacing, colors, and border-radius.
Never use arbitrary values like p-[13px].
```

### Ошибка 5: Забывать про состояния
**Проблема**: UI ломается при загрузке, ошибках, пустых данных

**Решение через системный промт**:
```
Every component must handle:
- Loading state (skeleton or spinner)
- Error state (with retry button)
- Empty state (with helpful CTA)
- Success state

Include all states in initial implementation.
```

### Ошибка 6: Desktop-only мышление
**Проблема**: UI красиво на 1920px, ломается на 375px

**Решение через промт**:
```
Mobile-first approach:
1. Design for 375px first
2. Add breakpoints for tablet (md:)
3. Add desktop enhancements (lg:)

Test at: 375px, 768px, 1280px
5. Практические примеры настройки MCP
Конфигурация для Cursor/Windsurf
.cursorrules или .windsurfrules:
markdown# Frontend Generation Rules

## Tech Stack
- React 18+
- TypeScript (strict mode)
- Tailwind CSS 3.4+
- shadcn/ui
- Lucide icons
- Framer Motion (optional)

## Visual Guidelines
- Spacing scale: 4, 8, 16, 24, 32, 48, 64px
- Typography scale: text-sm, text-base, text-lg, text-xl, text-2xl, text-3xl
- Color: HSL format, primary + accent + neutrals
- Shadows: subtle (shadow-sm, shadow-md)
- Transitions: 200ms ease

## Component Rules
1. Always start with shadcn/ui if component exists
2. One component = one file
3. Mobile-first responsive
4. Include all states (loading, error, empty, success)
5. Proper TypeScript types (no `any`)

## Anti-patterns
- ❌ Inline styles
- ❌ Custom CSS (use Tailwind)
- ❌ Arbitrary spacing values
- ❌ Missing hover states
- ❌ No mobile breakpoints

## Code Quality
- Descriptive names: `UserDashboardHeader` not `Header`
- Extract repeated JSX into components
- Use semantic HTML
- Add ARIA labels
- Comment design decisions

6. Чек-лист качества финального UI
Перед тем как считать UI готовым:
Визуальное качество

 Четкая визуальная иерархия (заголовки > подзаголовки > текст)
 Консистентные отступы (не смешаны p-4 и p-6 без причины)
 Ограниченная палитра (1-2 акцента + нейтральные)
 Читаемая типографика (min 16px для body text)
 Профессиональные тени (не черные drop-shadow)

Интерактивность

 Все кнопки имеют hover состояние
 Плавные переходы (transition-colors duration-200)
 Видимый фокус для клавиатурной навигации
 Disabled состояния выглядят отключенными
 Кликабельные элементы выглядят кликабельными

Адаптивность

 Работает на 375px (iPhone SE)
 Оптимизировано для 768px (tablet)
 Использует пространство на 1920px
 Нет горизонтального скролла
 Touch-friendly (min 44px tap targets)

Состояния

 Loading: skeleton или spinner
 Error: понятное сообщение + retry
 Empty: дружелюбное сообщение + CTA
 Success: визуальное подтверждение

Производительность

 Нет лишних re-renders
 Оптимизированы изображения
 Lazy loading для тяжелого контента
 Debounce для поиска/фильтров
