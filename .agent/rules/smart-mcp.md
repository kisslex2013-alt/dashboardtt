# Правила использования Smart Coding MCP

Вы должны приоритизировать использование инструментов **Smart Coding MCP** для следующих задач.

## 1. Управление зависимостями

**Когда:** При проверке, добавлении или обновлении версий пакетов (npm, python, go, rust и т.д.).

**Действия:**
- **ОБЯЗАТЕЛЬНО** используйте инструмент `d_check_last_version`
- **НЕ** угадывайте версии и не доверяйте внутренним данным обучения
- **НЕ** используйте обычный веб-поиск, если `d_check_last_version` не сработал

## 2. Исследование кодовой базы

**Когда:** При вопросах о том "как" что-то работает, поиске логики или понимании архитектуры.

**Действия:**
- **ОБЯЗАТЕЛЬНО** используйте `a_semantic_search` как ПЕРВЫЙ инструмент для любого исследования кодовой базы
- **НЕ** используйте `Glob` или `Grep` для разведывательного поиска
- Используйте `Grep` ТОЛЬКО для точного совпадения строк (например, поиска конкретного сообщения об ошибке)
- Используйте `Glob` ТОЛЬКО когда вы уже точно знаете шаблон имени файла

## 3. Окружение и статус

**Когда:** При начале сессии или отладке окружения.

**Действия:**
- Используйте `e_set_workspace` если текущий путь к рабочему пространству неверен
- Используйте `f_get_status` чтобы убедиться, что MCP-сервер работает и проиндексирован

## 4. Дополнительные инструменты

- `b_index_codebase` - ручная переиндексация проекта (если код значительно изменился)
- `c_clear_cache` - сброс всех данных и переиндексация с нуля (при проблемах)
