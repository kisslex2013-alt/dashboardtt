### 2. Строгая типизация

**Проблема**: Местами используется `any`, не всегда используются современные возможности TypeScript.

**Улучшения**:
```typescript
// 1. Строгие типы для API
interface APIResponse<T> {
  data: T
  meta: {
    timestamp: string
    version: string
  }
}

// 2. Использовать 'satisfies' (TS 4.9+)
const config = {
  apiUrl: 'https://api.example.com',
  timeout: 5000
} satisfies Config

// 3. Runtime валидация с Zod
import { z } from 'zod'

const TimeEntrySchema = z.object({
  id: z.string().uuid(),
  duration: z.union([z.string(), z.number()]),
  earned: z.union([z.string(), z.number()]),
  // ...
})

type TimeEntry = z.infer<typeof TimeEntrySchema>
```

**Action items**:
- [ ] Заменить все `any` на конкретные типы или `unknown`
- [ ] Добавить Zod для валидации данных из localStorage
- [ ] Использовать `satisfies` для констант
- [ ] Включить `strict: true` в tsconfig (если ещё не включено)

**Приоритет**: 🟡 Средний
