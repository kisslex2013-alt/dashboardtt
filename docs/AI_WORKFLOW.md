# AI-Assisted Development Workflow (Исследование №8)

На основе "Исследования №8: MCP для визуального фронтенда" — оптимальные воркфлоу и шаблоны промптов для Antigravity/Cursor.

## 🏆 Воркфлоу: "Сеньорный Промпт" (Скриншот → Код)

**Когда использовать**: Конвертация дизайна/скриншота в код.

**Шаг 1: Анализ (Без кода)**
> "Действуй как Senior Frontend Designer. Проанализируй приложенный скриншот. НЕ пиши код. Вместо этого:
> 1. Определи иерархию компонентов (контейнеры, сетки).
> 2. Извлеки дизайн-токены (цвета, шрифты, отступы).
> 3. Сопоставь элементы с **компонентами Shadcn/UI** (например, 'Это похоже на Card, вот это на Badge').
> 4. Выдай структурированный план."

**Шаг 2: Скелет**
> "Используя план выше, создай структуру страницы на семантическом HTML (<section>, <article>) и компонентах Shadcn. Не добавляй глубокую стилизацию — только структуру."

**Шаг 3: Стилизация (Визуальный проход)**
> "Теперь примени Tailwind CSS, используя извлечённые дизайн-токены. Для визуального эффекта используй компоненты **Magic UI** (например, Blur Fade) где уместно."

---

## 💎 Воркфлоу: "Визуальная Полировка" (Рефакторинг)

**Когда использовать**: UI выглядит "нормально", но скучно.

> "Действуй как эксперт по UI/UX. Сохрани текущую структуру, но улучши **Визуальную Иерархию**:
> 1. **Отступы**: Увеличь whitespace (используй `py-8` вместо `py-4`, `gap-6` вместо `gap-4`).
> 2. **Типографика**: Убедись, что заголовки минимум в 2 раза крупнее основного текста.
> 3. **Обратная связь**: Добавь hover-состояния (`hover:shadow-md`, `active:scale-95`) на все интерактивные элементы.
> 4. **Анимация**: Добавь лёгкие entrance-анимации через `framer-motion` (fade-in, slide-up).
>
> Используй `src/styles/tokens.css` для консистентных значений."

## 🧩 Шаблон Создания Компонента

**Когда использовать**: Запрос на новый компонент.

> "Создай [Название Компонента] со следующими требованиями:
> - **Технологии**: React + TypeScript + Tailwind.
> - **База**: Сначала проверь, есть ли компонент в Shadcn/UI (например, используй `Card` для контейнеров).
> - **Состояния**: Обязательно включи Loading (Skeleton), Error и Empty состояния.
> - **Мобильная версия**: Сначала дизайн для 320px, потом масштабирование.
> - **Доступность**: Включи ARIA-метки и клавиатурную навигацию."

## 🪄 Рекомендации по Magic UI

Для достижения "премиального" ощущения, упомянутого в исследовании, рассмотри эти компоненты:
- **Border Beam**: Для подсветки активных карточек.
- **Animated List**: Для ленты логов/истории.
- **Number Ticker**: Для статистики в аналитике.
- **Blur Fade**: Для последовательной загрузки изображений/контента.
