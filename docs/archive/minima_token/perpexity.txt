80% расхода токенов — это контекст, а не промты: Разработчики тратят огромное количество токенов на передачу файлов, истории чата и системных промтов. Сам запрос — часто менее 5% от общего объёма.

Выходные токены стоят 3-5x дороже входных: Все провайдеры применяют 3-5x премию за выходные токены. Оптимизация выхода часто эффективнее, чем входа.
​

Состояние чата множит стоимость экспоненциально: Каждое новое сообщение в одном чате переобрабатывает ВСЮ историю. 20 сообщений стоят в 18x раз дороже, чем одно.

3.2 Локализация Изменений
Принцип: Всегда передавай только релевантный код + 2-3 строки контекста.

typescript
// КОНТЕКСТ (необходимо)
export class UserService {
  constructor(private db: Database, private cache: Cache) {}

  // ЦЕЛЕВОЙ КОД (включи полностью)
  async getUser(user_id: string): Promise<User> {
    const user = await this.db.query(User).where({ id: user_id }).first();
    return user;
  }

  // СОСЕДНИЙ КОД (только сигнатуры)
  async updateUser(user_id: string, data: dict): Promise<User>
  async deleteUser(user_id: string): Promise<void>
}
3.3 Inline-Инструкции, TODO, Комментарии как Контекст
Стратегия: Оставляй TODO прямо в коде, а не отдельно.

typescript
// TODO: Add caching layer here
// Expected: Check cache first, then DB
// Impact: Reduce N+1 queries
// Deadline: This task
async getUser(user_id: string): Promise<User> {
  // OPTIMIZE: This query is called 50+ times per hour
  return await this.db.query(User).where({ id: user_id }).first();
}
Модель видит контекст сразу и работает точнее. Экономия: 20-30% на пояснениях.

class TokenBudget:
    def __init__(self, monthly_limit=50000):
        self.monthly_limit = monthly_limit
        self.used = 0
        
    def check_before_call(self, prompt_tokens, estimated_output):
        total = prompt_tokens + estimated_output
        if self.used + total > self.monthly_limit * 0.8:  # Alert at 80%
            suggest_cheaper_model()
        if self.used + total > self.monthly_limit:
            raise BudgetExceeded()
            
    def log_call(self, input_tokens, output_tokens, model, cost):
        self.used += input_tokens + output_tokens
        log_to_dashboard(model, cost, trending)  # Track which models blow budget

6.4 Trade-offs: Экономия Токенов vs Качество
Подход	Экономия Токенов	Качество	Best Use
Zero-shot (без примеров)	100% vs few-shot	70%	Простые задачи
Function sigs only	85% vs full code	85%	Рефакторинг
Diff format	90% vs full file	90%	Code edits
Few-shot (2 examples)	50% vs 5 examples	92%	Learning curve
Plan mode (Cursor)	60% vs max mode	95%	Complex builds
Haiku model	80% vs Sonnet	85% for simple	Low-complexity
Prompt caching	90% repeat reads	100%	Static context
Вывод: В 90% сценариев можно сэкономить 70-90% токенов без заметной потери качества.

ВЫВОДЫ
80% экономии достижимо без потери качества через правильное управление контекстом и выбор формата вывода (diff/signature-only).

Prompt caching — критический инструмент: 90% дешевле при повторном доступе к тому же контексту. Это базовый паттерн, а не оптимизация.

Выходные токены дороже входных в 3-5x: Оптимизация выхода (diff vs full file) часто выгоднее, чем входа.

Управление чатами = управление стоимостью: Каждое сообщение в одном чате множит стоимость. Новый чат per task (5 сообщений max) критичен.

Модели дорожают пропорционально полезности: Правильный выбор модели (70% Haiku, 30% Sonnet, 5% Opus) экономит 60-80% без ущерба качеству.

Финальное число: Разработчик, применяющий эти практики, может сократить расходы с $150-200/месяц до $30-50/месяц при лучшем или равном качестве выходов.
