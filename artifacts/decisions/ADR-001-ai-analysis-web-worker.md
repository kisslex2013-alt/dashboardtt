# ADR-001: Web Worker для AI-анализа

## Статус
Принято

## Контекст
AI-анализ (выгорание, прогнозы, аномалии) выполнялся синхронно в главном потоке каждые 30 минут через `useAINotificationMonitor`. При большом количестве записей (~1000+) это вызывало заметное "подвисание" UI.

## Варианты

### 1. Оптимизация существующего кода (in-place)
- ✅ Минимальные изменения
- ❌ Не решает проблему блокировки main thread
- ❌ Ограничен потенциал улучшения

### 2. Web Worker (выбран)
- ✅ Полностью освобождает main thread
- ✅ Масштабируется с ростом данных
- ✅ Чистое разделение логики
- ❌ Немного сложнее в поддержке (сериализация данных)

### 3. Debounce + requestIdleCallback
- ✅ Простая реализация
- ❌ Не гарантирует отсутствие блокировки
- ❌ Не подходит для тяжелых вычислений

## Решение
Выбран **Web Worker** как наиболее надежное решение для фоновых вычислений. Создан `src/workers/aiAnalysisWorker.ts`, который:
1. Принимает `entries`, `dailyGoal`, `categories` через `postMessage`.
2. Выполняет все анализы (burnout, forecast, anomaly и т.д.).
3. Возвращает агрегированные результаты.

Хук `useAINotificationMonitor` рефакторен для:
- Инициализации Worker через Vite's `new URL` синтаксис.
- Кэширования результатов по сигнатуре данных.
- Отслеживания `lastAnalyzed` timestamp.

## Последствия
- **Положительные**: UI остается отзывчивым при любом объеме данных.
- **Отрицательные**: Добавлена async-логика для обработки сообщений Worker.
- **Требует внимания**: При добавлении новых типов анализа нужно обновлять и Worker, и обработчик в хуке.
